# ダッシュボードデザインレビュー

このスキルは、デジタル庁の「ダッシュボードデザインの実践ガイドブック」とanemoの「ダッシュボードUIデザインガイドライン」に基づいて、ダッシュボードのデザインをレビューします。

画像・スクリーンショットだけでなく、**公開URLからも直接レビューできます**。

## 入力の種類

### パターン A: 画像・スクリーンショット
ユーザーがダッシュボードの画像を添付している場合 → そのままStep 1へ進む

### パターン B: URL（Looker Studio / Tableau Public / Metabase 等）
ユーザーがダッシュボードのURLを貼った場合 → 以下の手順でキャプチャしてからレビュー：

1. `mcp__claude-in-chrome__tabs_context_mcp` で現在のタブ状態を確認する
2. `mcp__claude-in-chrome__tabs_create_mcp` で新しいタブを作成する
3. `mcp__claude-in-chrome__navigate` でURLを開く
4. ダッシュボードのロードを待つ（必要に応じて数秒待機）
5. `mcp__claude-in-chrome__computer` でスクリーンショットを撮影する
6. ダッシュボードが縦に長い場合はスクロールして追加キャプチャする
7. キャプチャした画像をもとにStep 1へ進む

**URLレビューの注意点**:
- ログインが必要なページは自動レビュー不可（スクリーンショットを手動で貼り付けてもらう）
- インタラクティブなフィルターは初期表示状態でレビューする
- ロード時間がかかるダッシュボード（Tableau等）は十分に待ってからキャプチャする

## レビューの流れ

### Step 1: ダッシュボードの文脈を把握する

レビュー前に以下を確認する（不明な場合はユーザーに確認する）：
- **種類**: 提示型（概況把握・簡単な操作）か探索型（詳細分析・複雑な操作）か
- **目的（Why）**: なぜこのダッシュボードが必要か
- **対象ユーザー（Who）**: 誰が見るか、リテラシーレベルは
- **閲覧シーン（When/Where）**: いつ、どこで見るか（会議室のモニター、個人PC等）

### Step 2: ガイドラインに基づいて確認する

以下の参照ドキュメントのチェックリストを使ってレビューする：

- `references/checklist.md`: 総合チェックリスト（必ず参照）
- `references/digital-agency-guidelines.md`: デジタル庁ガイドラインの詳細
- `references/anemo-guidelines.md`: anemoガイドラインの詳細

### Step 3: フィードバックを出力する

## 出力フォーマット

```
## ダッシュボードレビュー結果

### 概要
（ダッシュボードの目的・種類の認識を1〜2文で確認）

### 良い点
- 具体的に良い点を列挙

### 改善点

#### 優先度: 高（ユーザー体験・情報伝達に直接影響）
1. **[カテゴリ]** 問題の説明
   - 現状: 何が問題か
   - 改善案: どう直すべきか
   - 根拠: ガイドラインのどの原則に基づくか

#### 優先度: 中（デザイン品質の向上）
...

#### 優先度: 低（細部の改善）
...

### チェックリスト結果
| カテゴリ | 項目 | 評価 |
|---------|------|------|
| 利用者体験 | ... | ○/△/× |
...
```

## 評価基準

- **○ (良好)**: ガイドラインに準拠している
- **△ (要改善)**: 部分的に対応しているが改善余地あり
- **× (要対応)**: ガイドラインに反している
- **N/A**: この項目は該当しない
