## 1. ダッシュボードの種類

### 提示型ダッシュボード（本ガイドブックの主対象）
- 概況の把握
- 一般的な知識で理解できる
- 単純な操作で利用できる

### 探索型ダッシュボード
- 詳細の分析
- 特定のドメイン知識が必要
- 複雑な操作を行う

## 2. 要件の整理

### 目的の定義（5W1H）

| 要素 | 内容 |
|------|------|
| **Why** | 最終的に達成したい目的、ダッシュボードを見る理由 |
| **What, so What** | 目的達成のために知るべき情報、見た後にする判断や行動 |
| **Who** | ダッシュボードを見る人、所属・立場・業務内容・リテラシーレベル |
| **When** | 見るタイミングや頻度 |
| **Where** | 見る場所、提示する媒体やデバイス |
| **How** | 求められる機能、必要なデータ項目・更新頻度 |

### 制約条件の整理

**ダッシュボード制作上の制約**:
- 情報の種類が多すぎる → 情報の種類を減らす
- 伝える内容が複雑すぎる → 複雑度を下げる
- 定期的に閲覧する必要がない

**データの制約**:
- データに欠損がある
- 更新頻度が不十分
- 必要な分解能がない
- 指標同士を比較できない

## 3. プロトタイピング

### 載せるべき情報を選ぶ4原則

1. **目的に則する**: 利用できるデータの中から、目的に当てはまるものを選び出す。そうでない情報はできる限り取り除き、データが判断や行動につながるまでの距離を縮める
2. **分解できる**: どの角度からデータを見るかを考え、それに必要なデータの分解粒度を最小単位とする。情報を全体から部分へと、見る人が読み解くことができる単位まで分割できるようにする
3. **違いに気づける**: データ全体の中で突出するデータに気づけるようにする。また、時間とともに変化する情報では、変化の時期と傾向（上昇や下降）に気づけるようにする
4. **鮮度が高い**: 利用可能な最新のデータをダッシュボードに反映し、より現時点に近い情報に基づいて意思決定ができるようにする

### ダッシュボード設計の4原則

1. **構造を伝える**: 全体から部分へと、階層構造になるようにダッシュボードの情報を整理する。最初に全体を示す情報を得られ、次にその詳細に辿り着けるように、レイアウトや導線を設計する
2. **適切な情報量にする**: 過多な情報によって、継続的な利用意欲が削がれることのないように配慮する。見る人にとって負担にならないように、情報の総量を調整する
3. **複雑な操作を要求しない**: 操作をせずに、あるいは少ない操作回数によって、必要な情報を得られるようにする。繰り返し利用することが、見る人にとって負担にならないようにする
4. **比較対象を提供する**: 数量単体では意思決定することが難しいデータには、目標値や平均値、前年の値を併記したり、前年比によってデータを表現するなど、比較できる対象を提供する

### レイアウトの考え方

- 視線の動きに合わせて**左上から右下**に向かって、全体から部分が並ぶように配置
- 左上: 全体を端的に示す情報（最も目を引く）
- 右下: 最も詳細な情報（判断や行動に役立つ細かい指標）
- 背景色に差をつけ、左上が最も目を引くように調整
- フィルターは画面の上部か左部に配置、影響を受ける情報はその下か右に配置
- **標準サイズ**: 1920×1080px（16:9）、縦横6分割のレイアウトグリッド

## 4. グラフの種類と選び方

| グラフ名 | 値の表現 | 伝える情報 | 使用場面 |
|---------|---------|-----------|---------|
| **折れ線グラフ** | 位置 | 時間変化、傾向 | 数量の時間変化を伝えたいとき。横軸は必ず時間 |
| **棒グラフ** | 長さ | 数量比較 | 分類毎の数量を比較するとき |
| **面グラフ** | 位置・面積 | 時間変化・傾向・構成比 | 複数系列を積み上げて使用。構成比の時間変化に着目するときは100%積み上げ |
| **円グラフ** | 角度・面積 | 構成比 | 全体の総量が明らかで構成比をコンパクトに伝える場合。多くは棒グラフの方が正確 |

## 5. カラーパレット

### 基本方針
- グラフで使用する色が**1〜3色の場合**: 同色の濃い明度を選ぶ
- グラフで使用する色が**4〜5色の場合**: 全ての明度から選択
- アクセシビリティのため**背景色とグラフ色面のコントラスト比を3:1以上**にする

### カラーシステム（デジタル庁デザインシステム）
推奨カラーファミリー: Blue, Light Blue, Cyan, Green, Lime, Yellow, Orange, Red, Magenta, Purple

### テキスト・背景の配色（Blueカラーパレット基準）
| 用途 | 色 | カラーコード |
|------|-----|-------------|
| Body（本文） | 黒 | #1A1A1A |
| Description（補足） | グレー | #626264 |
| On Fill（塗り上テキスト） | 白 | #FFFFFF |
| Link | ブルー | #0017C1 |
| Positive Number（正の数値） | ブルー | #0031D8 |
| Negative Number（負の数値） | 赤 | #FA0000 |
| Primary Background | ブルー | #0031D8 |
| Secondary Background | ライトグレー | #F8F8FB |
| Positive Background | 薄いブルー | #E8F1FE |
| Negative Background | 薄いピンク | #FDEEEE |

### コントラスト比の確保
- グラフ色面に対して背景色とのコントラスト比 **3:1以上**
- 数値テキストの色は背景色に対して **4.5:1以上**
- Blueカラーパレットの場合: **Blue-500（#4979F5）以上の濃さ**で配色

### 色覚多様性への対応
- 色以外の情報（データ点の形、グラフの塗りの模様）でも系列を識別できるようにする
- カラーユニバーサルデザイン推奨の配色を使う
- 色覚特性シミュレーションやチェッカーで確認する
- 白黒印刷しても伝わる情報となるよう意識する

## 6. グラフ設計の原則

### 知りたいことを知れる

1. **シンプルにする**: 冗長なデータや説明、過度に装飾的な表現など、データを理解するために不必要な情報をできるだけ取り除く
2. **意味のある順列にする**: 必要な情報を見つけやすいように配慮してデータを並べる（数量の大小順、アルファベット順など）
3. **強弱をつける**: 重要な情報には、それ以外とは異なる表現（色やサイズ、文字の太さなど）を用いる
4. **待たせすぎない**: 素早く情報を得られるようにするために、必要な情報を読み込み、画面に表示するまでにかかる時間を短くする

### 誤解を生まない

1. **わかりやすく表記する**: 前提知識が必要とされる表現や複雑な文言を避け、タイトルや系列名、数値の単位を丁寧かつ簡潔に表記する
2. **データを定義する**: 何を対象として収集したデータで、数値の意味が何であり、いつ更新されたかといった、提供するデータの定義を参照できるようにする
3. **表現を歪曲しない**: グラフ軸の値範囲を恣意的に設定しない。値の範囲を省略しない。数量の差が実際よりも誇張や矮小化されて見えるような表現をしない
4. **メタ情報を記載する**: ダッシュボードに使われているデータのソース、データの更新日、いつ時点の数値なのか、注釈、免責事項など、見る人によって解釈の幅がないように補足情報を提供する

## 7. Do's & Don'ts

### 知りたいことを知れる

| Do | Don't |
|----|-------|
| 全体の指標と詳細のグラフを表示する | 詳細の要素だけが並んでいる |
| グラフ項目の並び順に意味を持たせる（数量順など） | 意味もなくあいうえお順で並んでいる |
| 不要な要素は削除する（グリッドや枠線を最小限に） | 目盛線やタイトル、小数部などが必要以上に記載されている |
| 不要な装飾やリッチな表現は使わない | 数値とは関係のない表現が多用されている（3D円グラフなど） |
| グラフに使用する色数を絞る | 注目してほしいグラフがどれかわからない（多色使い） |

### 誤解を生まない

| Do | Don't |
|----|-------|
| タイトルにグラフの内容とデータ種別を表記する（「国産自動車の出荷台数（月次推移）」） | 何を表すグラフかすぐに分からないタイトル（「国産自動車」） |
| タイトルや凡例をシンプルに保つ | 冗長なタイトルや凡例 |
| グラフと凡例を隣接させる、凡例の順序をグラフと対応づける | 凡例が隣接していない・凡例の順番がグラフと逆 |
| 棒グラフの原点は原則として0にする | 原点が0になっていない（差が誇張・矮小化される） |
| 色のみで分類を識別しない（形や模様も使う） | 違うグラフで同じ形のマーカーを使っている |
